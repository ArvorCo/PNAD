# Code of Conduct

## Our Standard

This project expects direct, technical, respectful collaboration.

Examples of behavior that contribute to a healthy community:

- being precise and evidence-driven
- criticizing ideas and implementations rather than people
- documenting assumptions and tradeoffs
- helping improve reproducibility, safety, and clarity
- accepting review feedback without turning discussion into ego theater

Examples of unacceptable behavior:

- harassment, personal attacks, or discriminatory language
- deliberately misleading claims about data, methodology, or results
- publishing private credentials, private datasets, or other sensitive material
- low-effort spam, marketing dumps, or AI slop PRs
- hostile behavior toward maintainers, contributors, or users

## Scope

This Code of Conduct applies in:

- GitHub issues
- pull requests
- discussions
- review comments
- project communication spaces controlled by maintainers

## Enforcement

Maintainers may remove comments, close threads, reject contributions, or ban
participants who violate this Code of Conduct.

## Reporting

For conduct or safety concerns, open a private security-style report if exposure
would be harmful, or contact the maintainers through the repository security
channel if configured.
